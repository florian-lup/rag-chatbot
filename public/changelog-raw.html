New features, fixes, improvements and polish to the Anara platform

August 12, 2025

Major model upgrades, improved citations, and file tagging in the editor

In the last few days we added support for several new models, including OpenAI's GPT-5 and GPT-OSS 120B. GPT-5 offers significantly more intelligent responses and a more expressive writing style. OSS is fast and reliable - we’re so impressed with its performance vs Gemini Flash 2.5 that we made it the default model for free users.

We also introduced GPT o3 and o4-mini with real-time “thinking process” and made major upgrades to existing models: Gemini 2.5 Flash now uses its advanced reasoning engine, also displaying thinking in real-time.

The citation hover component in chat is now significantly more insightful. Instead of only showing the raw text from a source, we now generate an explanation of why that particular citation is relevant to the conversation. This summary, along with key quotes from the document, makes it much easier to verify the AI's reasoning without having to open and read the entire source document.

Improvements

  • You can now tag files and folders directly inside the editor. Simply type '@' to search for and link to any item in your library.
  • The @Search Web and @Search Papers agents are now significantly faster and more accurate, as we've refined the way we process search queries.
  • Added a copy transcript button for audio recordings. Click the three-dot menu next to the speed controls to copy the transcript.
  • We've polished the text streaming animation. Responses from AI models now appear much more smoothly as they are generated.
  • Added support for the Thai language in chat and editor AI generations.
  • Our agents can now intelligently use multiple tools to answer a single question. For example, an agent can now search the web and your private documents in the same step to create a more comprehensive response.
  • You can now attach PDF documents directly to chat messages.
  • The table view now automatically resets any active filters after you import new items, ensuring new imports are immediately visible.

Bugs

  • Fixed an issue where the same filter could be applied to a field multiple times in the table view.
  • Fixed an issue where in some cases deleting a folder from the sidebar would not delete the files and chats contained within it.
  • Resolved a bug where the chat input would sometimes be disabled for notes that contained content.
  • Fixed an issue where multiple sets of default folders were created for new users during the onboarding process.
  • Fixed a bug where pressing the Return key in the text editor slash command menu would create a new line instead of selecting the highlighted option.
  • Fixed an issue on mobile devices where a banner could hide the support button.
  • Resolved an issue where opening the Command+K menu on very large workspaces could cause the app to crash.

July 14, 2025

Citation counts, left sidebar updates and more table customization

This week's release focuses on refinement - adding OSCOLA citation support for legal researchers, bringing citation counts front and center, and giving you more control over your table view with customizable metadata columns.

We also reorganized the left sidebar for better chat access and fixed several bugs.

Improvements

  • Added support for the OSCOLA (Oxford University Standard for Citation of Legal Authorities) referencing format. This is now available as an option when generating citations and bibliographies.
  • A citation count for each document now appears in the main table view and in the right sidebar. This allows you to quickly see how many references are associated with a file without needing to open it.
  • All metadata fields that were previously only visible in the right sidebar can now be added as columns to the main table view. This provides more flexibility to customize your layout and compare key document information across all your files at a glance.
  • We've updated the left sidebar to include a new, dedicated section for your chats. It displays your five most recent conversations for quick access and can be expanded to view your full chat history.
  • The left sidebar is now also optimized for smaller screens and this update includes several bug fixes for a more stable experience.

  • In addition to your personal Zotero library, you can now import documents directly from your shared group collections.

Fixes

  • Resolved a layout bug on mobile devices where the 'Share' container would not fit on the screen correctly.
  • Resolved an issue with the ++ command for generating text in the editor. The feature would sometimes delete the first word or the first few letters of the AI-generated content immediately after inserting it, which will no longer happen.
  • Resolved a styling issue in the note sideview that could cause the title of a document to overlap with the content below it.
  • Fixed a bug that caused the "Published" column in the table view to always appear empty.
  • Resolved a bug in the document viewer that caused mathematical equations in Markdown files to render incorrectly.
  • Fixed a layout bug on mobile devices that caused the Import menu and its buttons to be cut off or pushed out of view.
  • Fixed an issue where importing from arXiv would fail if the provided link was missing the https:// prefix.
  • Fixed a rendering bug that occurred when opening the website from the Twitter app, which could cause parts of the page to be unreadable in dark mode.
  • Fixed an issue that prevented import limits from increasing correctly for new users
  • Fixed a bug where some AI-generated content, such as Notes, would be in English even when a different language was selected in settings.

June 2, 2025

Introducing Table View

Today, we’re super excited to share the next major iteration of Anara - Table View.

Anara started as a way to quickly understand documents. Over time we evolved, first with a powerful text editor, then with support for diverse media like video and audio.

As your libraries grew, we saw the need for a better way to organize everything.

That’s why we built Table View.

Table View is your new command center - a dynamic, spreadsheet-like interface designed specifically for scientific content. It offers a flexible and organized way to see, sort, filter and discover information across your library.

One of the most powerful features of Table View is contextual chat. You can now chat with any file or folder by opening the chat interface at that level.

All conversations are contextually stored - for example, if you chat with a specific folder, those conversations will be preserved within that folder, making it easy to resurface past insights.

Improvements

  • You can now edit previous message in the chat by clicking on the previously sent message.
  • Improved bulk import workflow to display a fresh upload list for each new import session, preventing confusion from previous session files
  • The items in the upload list are also now clickable making it easier to navigate to recently imported items.

Fixes

  • Fixed an issue where chat history was not properly maintained when chatting with library, ensuring conversation context is preserved.
  • Resolved an issue where AI messages would display as permanently loading in some cases when cancelled or interrupted.
  • Fixed an issue where some teams users experienced problems interacting with documents, ensuring proper access and functionality across all team workspaces.
  • Improved table formatting in chat to properly display bullet points as structured lists instead of showing HTML markup in generated content.
